## 1. What is OpenAPI 3?
OpenAPI Specification (OAS) is a standard, language-agnostic interface description for RESTful APIs. It allows both humans and computers to discover and understand the capabilities of a service without access to its source code.

---

## 2. OpenAPI vs Swagger
- **OpenAPI**: The specification (the rules).
- **Swagger**: The suite of tools (Swagger UI, Swagger Editor) used to implement and visualize the specification.

---

## 3. SpringDoc OpenAPI
The `springdoc-openapi` library is the modern successor to SpringFox. It automatically generates the OpenAPI JSON/YAML and the Swagger UI by inspecting your Spring Boot application at runtime.

---

## 4. Documenting Controllers
- `@Tag`: Used to group related endpoints (e.g., "User Management").
- `@Operation`: Describes what a specific method does (Summary and Description).

---

## 5. The Classic Interview Trap: Leaking APIs
**The Trap**: Enabling Swagger UI in the **Production** environment.
**The Problem**: Attackers can use your Swagger UI to easily map your entire API surface, find hidden "internal" endpoints, and understand your data models.
**The Fix**: Only enable Swagger in `dev` or `test` profiles using Spring Profiles, or protect the `/swagger-ui.html` path with Spring Security.

---

## 6. Documenting Models
Use `@Schema` on your DTOs to provide clear descriptions, example values, and to indicate which fields are required.
```java
@Schema(description = "User details")
public class UserDto {
    @Schema(example = "john_doe")
    private String username;
}
```

---

## 7. Parameters and Responses
- `@Parameter`: Documents individual query params or path variables.
- `@ApiResponse`: Documents the possible HTTP status codes (200, 401, 404) and the data they return.

---

## 8. Grouping APIs
If you have a large application, you can create multiple "Groups" in Swagger (e.g., "Public API", "Admin API", "Internal API") to make the documentation easier to navigate.

---

## 9. Security Configuration
To allow users to test protected endpoints directly from Swagger UI, you must configure a `SecurityScheme` (e.g., Bearer JWT) in your OpenAPI bean.

---

## 10. Client Generation
The `openapi.json` file generated by Spring can be used with tools like `openapi-generator` to automatically create client-side libraries in TypeScript, Python, or Go that perfectly match your API.

---

## 11. Common Mistakes
1. Not providing examples (makes the UI hard to use for testers).
2. Forgetting to document error responses (400, 500).
3. Over-documenting (keep descriptions concise and meaningful).

---

## 12. Quick-Fire Interview Q&A
**Q: How do you access the raw OpenAPI JSON?**  
A: Usually at `/v3/api-docs`.  
**Q: Can you hide a specific controller from Swagger?**  
A: Yes, use the `@Hidden` annotation.
